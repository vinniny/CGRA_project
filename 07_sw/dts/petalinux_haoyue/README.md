## Why HDMI IPs are NOT bound by Linux drivers

The Xilinx in-tree drivers for `xlnx,axi-vdma-6.3` and `xlnx,v-tc-6.2` exist
but they need DRM/DMABUF wiring that is heavy and version-sensitive. On
Haoyue the `system-user.dtsi` keeps the original compat strings but the
userspace `hdmi-demo` app skips the kernel binding entirely and pokes the
registers via `/dev/mem`. This is the proven path — replicate it on PYNQ-Z2.
